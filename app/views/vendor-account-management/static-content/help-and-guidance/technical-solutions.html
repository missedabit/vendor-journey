Technical solutions

Phase 1: Planning your API configuration

To manage the process for redemption of grant tokens, Application Programming Interfaces (APIs) will be used to pass information between HtG:D and your website. Your organisation will be required to configure three APIs to manage redemption of grant tokens.

Information for your software developer

You will receive a separate Vender Developer Guidance in w/c 11th October detailing API specifications and Swagger documentation. If you need to engage a third-party develop to configure this for you, you can share the document. You will receive your access code and stub URLs with access to the Vendor Portal from the w/c 18th October.

The three APIs are:

1. Voucher Check API

When your customer visits the redemption link, grant token details will be provided to this API. The API can authorise that the customer can redeem the grant token and discount.

2. Redeem Update API

When your customer completes their transaction, this API will mark the grant token as used.

3. Reconciliation API

You will need to call this API daily to report all HtG:D eligible sales to the grant administrator.
